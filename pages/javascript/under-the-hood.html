JavaScript Engine


JavaScript Engine là một chương trình thực thi mã nguồn (Interpreter) được viết bằng một phiên bản tiêu chuẩn của ngôn ngữ ECMAScript. Vì ECMAScript (ES) là đặc tả được tiêu chuẩn hóa(standardized specification) của JavaScript nên ECMAScript Engine là một tên gọi khác của JavaScript Engine.

Các JavaScript Engine thường được phát triển bởi các nhà cung cấp trình duyệt. Trong trình duyệt, JavaScript Engine chạy phối hợp với Rendering Engine thông qua DOM(Document Object Model).

Hiện nay, việc sử dụng các JavaScript Engine không còn bị giới hạn ở các trình duyệt. Như V8 là một JavaScript Engine được phát triển bởi Google, được sử dụng trong trình duyệt Chrome và cả trong Node.js(Runtime System).

Javascript is a single threaded single concurrent language - ngôn ngữ đồng thời đơn luồng. Điều này có nghĩa là JavaScript chỉ có thể xử lý 1 tác vụ (1 dòng code) tại 1 thời điểm. JavaScript có Call Stack, Heap và Queue tạo thành JavaScript Concurrency Model (Mô hình được sử dụng trong V8 Engine).


⚫ Call Stack

Call Stack là một cấu trúc dữ liệu được dùng để lưu lại "function calls". Thông qua Call Stack, chúng ta có thể biết được dòng script nào đang được thực thi.

Chúng ta có thể thấy rõ cách Call Stack làm việc bằng cách Throw Error

Đôi khi, chúng ta gặp phải vòng lặp vô hạn. Đối với trình duyệt Chrome, Call Stack có giới hạn về kích thước là 16.000 frames, nếu có nhiều hơn số frame tối đa, chúng ta sẽ gặp lỗi Max Stack Error Reached
