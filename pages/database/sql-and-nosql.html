What are differences between SQL and NoSQL ?

SQL No SQL
Data Storage Model
Predefined Schema (number, name and data type of columns are fixed).
- Support Premitive and Document data types
- Dynamic Schema (data is unstructured)
- Support more complex data types
+ Document (JSON, XML)
+ Key-value
+ Wide columns (Table with rows and dyamic column)
+ Graph
Primary purpose General Purpose
Document: General purpose
Key-value: Large amounts of data with simple lookup queries
Wide-column: Large amounts of data with predictable query patterns
Graph: Analyzing and traversing relationships between connected data
Scaling Vertical Scale (Scale up) Horizontal Scale (Scale out)
Hierarchical data storage Not suitable for hierarchical data storage More suitable for the hierarchical data store as it supports key-value pair method.
Relational Supported Non Relational (Distributed Database)
Model ACID Base (Basically Available, Soft state, Eventually Consistency) is a model of many NoSQL systems
ACID Transactions Supported Most do not support. But some still support.
Consistency Strong Consistency Eventual Consistency
Joins Typically required Typically not required
ORM Require Required Many do not required

Why SQL is hard to scale out ?

Vì gần như tất cả các RDBMS đều tuân thủ ACID.

Trong môi trường Distributed Database, việc đảm bảo tính đúng đắn của dữ liệu thì cần phải có một kênh giao tiếp phụ trợ nhanh để đồng bộ hóa dữ liệu giữa các server.

Điều này sẽ trở nên khó khăn hơn rất nhiều khi chúng ta có thêm nhiều server.

Vì càng cập nhật cho nhiều server thì càng tốn nhiều thời gian. Lúc này ta sẽ đối mặt với bài toán performance.


Why NoSQL is easy to scale out ?

Vì đa phần các NoSQL Database chỉ hỗ trợ Eventual Consistency. Điều đó khiến cho NoSQL không nhất thiết phải đảm bảo tính đúng đắn của dữ liệu khi trả về.

Việc này khiến NoSQL đơn giản hơn rất nhiều trong môi trường Distributed Database


When should we use SQL instead of NoSQL ?

- Khi chúng ta cần thể hiện rõ ràng mối quan hệ các dữ liệu (như trong report hoặc phân tích dữ liệu).

- Khi chúng ta không thể chấp nhận sự sai sót mà cần đảm bảo tính đúng đắn và toàn vẹn của dữ liệu.

- Khi sự tăng trưởng về số lượng cũng như kiểu dữ liệu không quá nhiều và có thể dự đoán được.
