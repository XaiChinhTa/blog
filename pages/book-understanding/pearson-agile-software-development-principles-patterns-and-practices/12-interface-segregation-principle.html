Meaning

If the Dependency Inversion Principle guides us that all objects should depend on the abstraction.

So the Interface Segregation Principle help us to organize and design that abstraction.

The principle makes 3 points:

Each client has each own interface.

The client is something uses that interface. It can be an another class, program, ...

Because the interface is owned by the client. So clients should not be forced to depend on methods that they do not use.

When the client is seperated. The interface should be seperated too. Because clients exert forces on the interfaces they use

When the interface changes, it effects on its clients. And the client can make its interface change.

The ways to seperate the interface.

Separation through Delegation

Separation through Multiple Inheritance (Prefer to use)

Polyadic form is refer than Monadic form

Sometimes, the methods invoked by different clients or group of clients will overlap. If the overlap is small, then the interfaces for that groups or clients should remain seperate.

When we maintains a app and we need to change the interfaces. If that change have a huge impact, the impact should be migrated by adding new interfaces to existing objects, rather than changing the existing interfaces.
