Every module or code block always has 3 functions. While one function of all function is broken, it needs to be refactored even though it works.

The reason of the exist of the module of code block. What it performs while executing.

Afford to change: When a module or code block is not simple and possible to change or extend, it is broken.

Communicate with its readers. This function helps the developers who unfamiliar with the module should be able and easy to read and understand. When the module or code block is too hard to understand, it's broken.

In the factoring process, we might face to extracting a big function to some small functions.

In some cases, that functions is called only once time that affects the performance. We will use perfomance and pain test to consider we should extract function or not.
